# 安全政策

## 报告安全问题

如果你发现本项目存在安全漏洞，请通过以下方式报告：

### 📧 联系方式

由于本项目目前还没有专门的安全邮箱，请直接通过 GitHub **Private Vulnerability Reporting** 功能提交报告。

### 📝 报告指南

为了提高处理效率，请在报告中包含以下信息：

1. **漏洞类型**（如：敏感信息泄露、代码注入等）
2. **受影响版本**
3. **复现步骤**
4. **潜在影响**
5. **修复建议**（如有）

## 当前支持的版本

| 版本 | 支持状态 |
| ---- | -------- |
| v1.0.x | ✅ 支持 |

## 响应流程

1. 收到报告后，我们将在 **72 小时内** 确认并回复
2. 确认后，我们会评估影响范围并制定修复方案
3. 修复完成后，会发布新版本并通知报告者

## 安全注意事项

### ⚠️ API 密钥管理

本项目需要配置阿里云 DashScope API Key，请注意：

- **不要** 将 `.env` 文件提交到 Git 仓库
- **不要** 在公开场合分享你的 API Key
- 定期检查 API Key 使用情况，发现异常及时更换

### 🔒 数据安全

- 本工具处理的音频、图片、视频等文件仅存储在本地
- 所有数据通过官方 DashScope SDK 传输至阿里云服务器
- 请勿上传涉及隐私或敏感信息的文件
